一，对数据规模建立一定认知

我们可以写段代码来测试一下某个代码段所花的时间。

xxxxxxxxxx

    }

根据实验而出结论，如果想要在 1 s 之内解决问题:

  • O(n^2) 的算法可以处理大约10^4级别的数据；
  • O(n log n) 的算法可以处理大约10^7级别的数据;
  • O(n) 的算法可以处理大约10^8级别的数据；

但是为了保险，我们一般会降低2倍，甚至降低一个数量级，来应对常数比较大的情况。

 

对于空间复杂度呢？

  • 多开常数空间: O(1)
  • 多开一个辅助的数组: O(n)
  • 多开一个辅助的二维数组: O(n^2)

double pow(double x, int n){
    assert(n >= 0);
    if(n == 0)
        return 1.0;
    double t = pow(x, n/2);
    if(n % 2)
        return x * t * t;

}

上面两个函数中只可能进入某一个递归调用，递归深度为log n，每次执行递归函数所花时间为 O(1)，因此总的时间复杂度为O(log n) 。

 

O(n)如下:

xxxxxxxxxx

上面的连加函数递归深度为 n ，每次执行递归函数所花时间为 O(1)，因此总的时间复杂度为O(n) 。

 

O（n log n)如下:

xxxxxxxxxx
void mergeSort(int arr[], int left, int right){
    if(left >= right)
        return;
    int mid = left + (right - left) / 2;
    mergeSort(arr, left, mid);
    mergeSort(arr, mid + 1, right);
    merge(arr, left, mid ,right);
}

明显上述是一个归并排序算法，递归深度log n ,每层处理的时间越来越小(偏离树根)，递归树上每个节点处理的时间为O(n)。因此它的时间复杂度为O（n log n )。复杂的分治算法可以去参考 主定理 。

 

我们发现上面的递归函数只进行一次递归调用，那么多次递归调用的情况呢？

斐波那契函数:

xxxxxxxxxx
int f(int n){
    assert(n >= 0);
    if(n <= 2)
        return 1;
    return f(n - 1) + f(n - 2);
}

这个时候我们应该注重计算调用的次数。可以画个递归树。

调用次数: 2^0 + 2^1 + 2^2 + 2^3 + ... + 2^n = 2^(n+1) - 1 = O(2^n)

所以上面是O（2^n)。当n = 20,计算量就上百万。当n = 30，计算机就很勉强了。

显然我们可以用动态规划去将上述指数级算法转换成多项式级别的算法。

我们看一下 这道题目

283 Move Zeroes. 给定一个数组 nums，写一个函数，将数组中所有的0挪到数组的末尾，而维持其它所有非0元素的相对位置。 比如,nums = [0,1,0,3,12], 函数运行后为[1,3,12,0,0]

先来看一下这朴素的想法

xxxxxxxxxx
void moveZeros(vector<int>& nums){
    vector<int> nonZeroElements;
    for(int i = 0; i < nums.size(); i++)
        if(nums[i])
            nonZeroElements.push_back(nums[i]);
    //把非0元素挪回去
    for(int i = 0; i < nonZeroElements.size(); i++)
        nums[i] = nonZeroElements[i];
    for(int i = nonZeroElements.size(); i < nums.size(); i++)
        nums[i] = 0;
}

时间复杂度：O(n)

空间复杂度: O(n)，多开了一个辅助数组。

 

我们思考一下，其实没必要多开一个辅助数组。我们可以借助常数空间来完成。

xxxxxxxxxx
void moveZeroes(vector<int>& nums){
    //nums中，[0,k)的元素均为非0元素
    int k = 0;
    for(int i = 0; i < nums.size(); i++)
        if(nums[i])
            nums[k++] = nums[i];
    for(int i = k; i < nums.size(); i++)
        nums[i] = 0;
}

时间复杂度: O(n)

空间复杂度: O(1)

 

其实也可以用另一种思路，把非零元素放前面时，把零放后边。

xxxxxxxxxx
void moveZeroes(vector<int>& nums){
    int k = 0;
    for(int i = 0; i < nums.size(); i++)
        if(nums[i])
            swap(nums[k++], nums[i]);
}

上述代码还要考虑一种极端情况：如果数组所有元素都不为0呢？ 那么每次都交换是没必要的。我们可以加个判断条件。此时时间复杂度O(n)和空间复杂度O(1)其实没变。

xxxxxxxxxx
void moveZeroes(vector<int>& nums){
    int k = 0;
    for(int i = 0; i < nums.size(); i++)
        if(nums[i])
            if( i != k)
                swap(nums[k++], nums[i]);
            else
                k++;
}

75 Sort Colors

给定一个有n个元素的数组，数组元素的取值只有0,1,2三种可能。为这个数组排序。

我们可以使用计数排序的思路。

xxxxxxxxxx
void sortColors(vector<int>& nums){
    //统计0,1,2的个数
    int count[3] = {0};
    for(int i = 0; i < nums.size(); i++){
        //assert(nums[i] >= 0 && nums[i] <= 2)
        count[nums[i]]++;
    }
    int index = 0;
    for(int i = 0; i <= 2; i++)
        for(int j = 0; j < count[i]; j++)
            nums[index++] = i;
}

时间复杂度:O(n)

空间复杂度:O(1)

 

上面的算法扫描了两遍数组。那么我们尝试一遍解决问题。

我们参考一下三路快速排序的思想。

0.....................zero i............................two 2.............2

i 向右推进过程中

  • 如果 i 为1,那么继续向前
  • 如果 i 为2，那么和two--，a [two] 与 a [i]交换
  • 如果i 为0，那么zero++,a [zero]与 a [i]交换
xxxxxxxxxx
void sortColors(vector<int>& nums){
    //nums[0,zero] 都为 0
    int zero = -1;
    //nums[two, n-1] 都为2
    int two = nums.size();
    for(int i = 0; i < two; ){
        if(nums[i] == 1)
            i++;
        else if(nums[i] == 2)
            swap(nums[i], nums[--two]);
            //下一继续处理nums[i]
        else{
            //assert(nums[i] == 0);
            swap(nums[++zero], nums[i++]);
        }
    }
}

时间复杂度:O(n)

空间复杂度:O(1)

vector<int> twoSum(vector<int>& numbers, int target){

还可以用对撞指针的思想。

i ...................................................................j

  • a [i] + a [j] < target , i++
  • a[i] + a[j] > target, j--
  • a[i] + a[j] == target, 找到了答案
xxxxxxxxxx
vector<int> twoSum(vector<int>& numbers, int target){
    int l = 0, r = numbers.size() - 1;
    //题目要求两个数，所以l = r不用考虑
    while(l < r){
        if(numbers[l] + numbers[r] == target){
            int res[2] = {l + 1, r + 1};
            return vector<int>(res, res + 2);
        }
        else if(numbers[l] + numbers[r] < target)
            l++;
        else
            r--;
    }
    //throw invalid_argument("The input has no solution.")
}

空间复杂度: O(1)

209 Minimum Size Subarray

给定一个整型数组和一个数字s，找到数组中最短的一个连续子数组，使得连续子数组的数字和sum >= s, 返回这个最短的连续子数组的返回值

比如，给定数组[2,3,1,2,4,3],s = 7 那么答案为[4,3], 返回2

暴力解: 遍历所有的连续子数组[i, j] , 计算其和sum, 验证sum >= s，时间复杂度O(n^3)，太大了！

思考一下暴力解的问题: 大量的重复计算。比如a [i, j-1]与 a [i, j]之间。

i.....j，i向右过程，sum为索引为 i 到 j 的 j - i + 1 个元素的和。

  • sum < s , j++,sum = sum + a [j]
  • sum > s, sum = sum - a [i] , i++
  • sum = s, 与原先的保存值比较，满足条件存一下窗口长度(j - i + 1)
xxxxxxxxxx
int minSubArrayLen(int s, vector<int>& nums){
    //nums[l,r]为我们的滑动窗口
    int l = 0, r = -1;
    int sum = 0;
    //赋一个大到不可能的值给res即可
    int res = nums.size() + 1;
    while(l < nums.size()){
        if(r + 1 < nums.size() && sum < s) 
            sum += sum[++r];
        else
            sum -= sum[l++];
        if(sum >= s)
            res = min(res, r - l + 1);
    }
    if(res == nums.size() + 1)
        return 0; //没有解
    return ret;
}

时间复杂度: O(n)

空间复杂度: O(1)

 

看一个难一点的题目:

3 Longest Substring Without Repeating Characters 在一个字符串中寻找没有重复字母的最长字串 如 "abcabcbb", 则结果为 ”abc“ 如"bbbbb" ,则结果为"b" 如"pwwkew", 则结果为 "wke"

用滑动窗口(双索引的方法)

xxxxxxxxxx
int lengthOfLongestSubstring(string s){
    //用来记录是否窗口中已有此字符串
    int freq[256] = {0};
    //滑动窗口为s[l, r]
    int l = 0, r = -1;
    //记录最长字串
    int res = 0; 
    while(l < s.size()){
        if(r + 1 < s.size() && freq[s[r + 1]] == 0)
            freq[s[++r]]++;
        else
            freq[s[l++]]--;
        res = max(res, r - l + 1);
    }
    return res;
}
