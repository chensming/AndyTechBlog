  • 事件的冒泡: 子元素的事件会逐层传递给父元素

  • 阻止事件的冒泡：bind绑定的方式转变为catch去绑定，如 bindtap 改 catchtap。 catchtap 相当于 catch:tap

    data-*有两种设置方式，可以直接在HTML元素标签上书写

    xxxxxxxxxx
    <div id="test" data-age="24">
            Click Here
    </div>

    其中的data-age就是一种自定义属性，当然我们也可以通过JavaScript来对其进行操作，HTML5中元素都会有一个dataset的属性，这是一个DOMStringMap类型的键值对集合。

    xxxxxxxxxx
    var test = document.getElementById('test');
    test.dataset.my = 'Byron';
    xxxxxxxxxx
    这样就为div添加了一个data-my的自定义属性，使用JavaScript操作dataset有两个需要注意的地方。
    ​
    (1) 我们在添加或读取属性的时候需要去掉前缀 <font color=#00ffff>**data-***</font>，像上面的例子我们没有使用 ~~test.dataset.data-my = 'Byron';~~的形式。
    ​
    (2)  如果属性名称中还包含连字符(-)，需要转成 <font color=#f05b72>**驼峰命名方式**</font>，但如果在CSS中使用选择器，我们需要使用连字符格式。

 

二，云开发部分

1. 认识三大云开发基础技术支持

  • 云函数(nodejs): 为用户提供了 在云端运行代码 的能力，很方便获取用户信息，如获取用户openid，很方便获取小程序appid，生成分享图，调用腾讯云SDK，对数据库进行操作。

  • 云数据库(mongDB): 可以通过小程序直接调用云数据库，也可以通过云函数调用云数据库，实现对数据库的 数据增加 ，数据删除 ，数据修改 和 数据查询等操作。

  • 云函数(nodejs): 客户端 直接上传或下载文件，管理文件，上传文件，下载文件，分享文件 。还支持在云开发控制台中对我们管理的文件进行可视化的操作。

    注意： 调试基础库信息版本要在 2.2.3 以上

 

 

2. 云数据库

  • 1.云开发提供了一个JSON数据库，提供2GB免费存储空间

    微信云开发提供的数据库实际上是mongDB,一种非关系型数据库。下表我们来看一下关系型数据库和非关系型数据的区别。

**若频繁对数据库进行操作的话，文档型数据库系效率会更高些。

 

  • 2.数据类型

    • String: 字符串
    • Number: 数字
    • Object: 对象
    • Array: 数组
    • Bool: 布尔值
    • GeoPoint: 地理位置点(用经纬度来表示)
    • Date: 时间(精确到毫秒),小程序创建的时间是客户端的时间，并不是服务端的时间
    • Null: 相当于占位符

 

  • 3.云数据库权限管理

    仅创建者可写，所有人可读(文章) 仅创建者可写(私密相册) 仅管理段可写(商品信息) 仅管理段可读写(后台比较敏感的数据)

    xxxxxxxxxx

     即可解决问题。
      
    创建的云函数文件比如sum下面会有两个文件，一个是index.js，一个是package.json。<font color=#00a6ac>**index.js**</font>文件是云函数的<font color=#f05b72>**入口文件**</font>。<font color=#00a6ac>**package.json**</font>文件定义了当前云函数所需的各种<font color=#f05b72>**模块**</font>，以及云函数的<font color=#f05b72>**配置信息**</font>。

  • 例子1: 上传文件到云存储中，并存到云数据库中。

    答疑: 为何不存在云存储中(基本配置5G)，还要存在云数据库中(基本配置2G)？

    xxxxxxxxxx
              因为能对云存储的数据数据进行操作的，除了管理员，就是云函数了。
    然而通过云函数对云存储进行的操作形式比较少，没有云数据库对数据的操作方便。
    例如对云数据库基本操作，增删改查等。并且云数据库并不真正存数据，它存的都是一条一条的文档（json对）。

执行流程： (1)用户选择图片或拍照，将图片信息传给小程序 (2)小程序上传所选的图片到云存储中 (3)云存储返回给小程序fileID (4)小程序将fileID存储到云数据库

 

wxml文件中：

xxxxxxxxxx
<button bindtap="upload">上传图片</button>

js文件中：

xxxxxxxxxx
upload: function(){
    //选择当前相册或拍照一张图片
    wx.chooseImage({
        //选择图片个数，最多9张(多占图片涉及异步处理)
        count: 1,
        //源文件形式上传还是压缩形式上传
        sizeType:['original', 'compressed'],
        //当前文件来源：相册或相机
        sourceType: ['album', 'camera'],
        success(res){
            //tempFilePath可以作为img标签的src属性显示图片
            const tempFilePaths = res.tempFilePaths
            //打印上传图片临时路径
            console.log(tempFilePaths);
            //上传图片
            wx.cloud.uploadFile({
                //上传至云端的路径,这里用时间戳作为名字
                cloudPath: new Date().getTime() + '.png',
                //小程序临时文件路径,string
                filePath: tempFilePaths[0],
                success: res=>{
                    //返回文件 ID
                    console.log(res.fileID)
                    //在云数据库新建一个imag集合,然后。。。
                    db.collection('image').add({
                        data: {
                            fileID: res.fileID
                        }
                    }).then(res=>{
                        console.log(res);
                    }).catch(err=>{
                        console.error(err);
                    })
                },
                fail: console.error
            })
        }
    })
 }

xxxxxxxxxx
 js页面的page函数内添加
xxxxxxxxxx
downloadFile: function(event){
    wx.cloud.downloadFile({
        fileID: event.target.dataset.fileid;
        success: res=> {
            //返回文件临时路径
            console.log(res.tempFilePath);
            //保存图片到手机相册
            wx.saveImageToPhotoAlbum({
                filePath: res.tempFilePath,
                sucess(res) {
                    wx.showToast({
                        title: '保存成功'
                    })
                }
            })
        },
        fail:console.error
    })
 }
